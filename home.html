Sistema SavvyLink

Documentación Técnica v2.1

1. Descripción General del Sistema

SavvyLink es una plataforma integral de evaluación y gestión de riesgos que permite a las empresas evaluar los perfiles de riesgo de los clientes a través de flujos de trabajo automatizados e integraciones. El sistema maneja la creación de cuentas, gestión de usuarios, análisis de riesgos y proporciona un directorio público para la verificación de perfiles de riesgo.

2. Configuración Inicial y Creación de Cuenta

Este video muestra el proceso completo de configuración inicial, desde el acceso a la página de inicio hasta la activación de la cuenta. Es fundamental seguir cada paso para garantizar una configuración correcta del sistema.

2.1 Acceso a la Página de Inicio

Esta es la página de inicio protegida por contraseña. Aquí los administradores inician el proceso de creación de cuenta ingresando la contraseña de acceso. La interfaz muestra claramente los campos requeridos y las opciones de segmentos de negocio disponibles.
URL: https://savvylink.nilsdigital.com/inicio
Autenticación: Protegida por contraseña
Credenciales de acceso: SL2025

2.2 Proceso de Creación de Cuenta

Formulario de checkout donde se completan los datos de la nueva cuenta. Es crucial seleccionar correctamente el segmento de negocio ya que esto determinará las configuraciones automáticas que se aplicarán posteriormente. El sistema validará todos los campos antes de proceder.

Para crear una nueva cuenta:

  • Navegar a la página de inicio protegida por contraseña
  • Completar el formulario de checkout con la siguiente información:
    • Seleccionar segmento de negocio de las opciones disponibles
    • Ingresar nombre del cliente
    • Proporcionar dirección de correo electrónico de facturación
  • Enviar formulario de checkout para crear cuenta en el backend

2.3 Activación de Cuenta

Email de activación que recibe el administrador. Este paso es obligatorio por razones de seguridad. El enlace de activación tiene una validez limitada y debe ser utilizado lo antes posible. Una vez activada la cuenta, se inician automáticamente todos los procesos de configuración.
  • El sistema genera un correo de activación enviado a la dirección de correo de facturación
  • El administrador debe hacer clic en el enlace de activación en el correo para activar la cuenta
Nota de Seguridad: La activación de cuenta es obligatoria para el cumplimiento de seguridad

3. Flujo de Trabajo Post-Activación

3.1 Activación Automática del Flujo de Trabajo

Diagrama del flujo de trabajo automático que se activa tras la activación de la cuenta. El sistema detecta el código del segmento de negocio desde la URL y ejecuta el workflow "Agregar Administradores a la Ubicación y Actualizar Valores Personalizados". Este proceso es completamente automático y no requiere intervención manual.
  • Evento Disparador: Activación de cuenta
  • Detección de Parámetros: El sistema captura el código del segmento de negocio desde la URL
  • Nombre del Flujo de Trabajo: "Add Admins to Location and Update Custom Values"

3.2 Procesamiento Backend

  • Ejecución de Webhook: La activación desencadena webhook al sistema backend
  • Procesamiento de ID de Ubicación: El backend verifica y valida el ID de ubicación
  • Actualizaciones de Campos Personalizados:
    • Agrega valor del segmento de negocio al campo personalizado
    • Actualiza información de subcuenta para análisis de riesgo
  • Asignación de Administradores: Todos los administradores designados se agregan automáticamente a la subcuenta
  • Coincidencia en Base de Datos: El sistema compara el nombre de la subcuenta con los registros de la base de datos usando el ID de ubicación asociado

Este proceso automatizado garantiza que cada cuenta nueva esté configurada correctamente con todos los permisos y configuraciones necesarias según el segmento de negocio seleccionado. La automatización reduce errores humanos y acelera el tiempo de implementación.

4. Gestión de Usuarios

4.1 Agregar Usuarios del Cliente

Video tutorial que muestra cómo agregar nuevos usuarios al sistema. El proceso es intuitivo y garantiza que cada usuario sea asignado correctamente a su cuenta correspondiente con los permisos adecuados.
Formulario de adición de usuarios ubicado en Sitios → Pestaña Formularios → Vista Previa. Este formulario captura toda la información necesaria del usuario y activa automáticamente el proceso de asignación mediante webhook. Es importante completar todos los campos requeridos para evitar errores en el procesamiento.
  • Navegación: Ir a Sitios → Pestaña Formularios → Vista Previa
  • Completar Formulario: Llenar el formulario de información de contacto para nuevos usuarios
  • Procesamiento Backend: El envío del formulario activa webhook al backend
  • Asignación de Usuario: El sistema asigna automáticamente al usuario a la cuenta correcta

4.2 Notificaciones Automatizadas

Configuración del webhook que se activa cuando se agrega un nuevo usuario. Este webhook envía la información a la cuenta de administrador de SavvyLink para procesamiento adicional y notificaciones. El sistema valida todos los datos antes de enviar el webhook para garantizar la integridad de la información.

Activador de Webhook: La adición de usuario envía webhook a la cuenta de administrador de Savvy Link

Sistema de notificaciones automatizadas que incluye WhatsApp y correo electrónico. Cada nuevo usuario recibe instrucciones detalladas sobre cómo acceder y utilizar el sistema. Las notificaciones son personalizadas según el tipo de cuenta y los permisos asignados.
  • Integración WhatsApp: El sistema envía mensaje automatizado de WhatsApp con instrucciones
  • Notificación por Email: Email automatizado enviado con instrucciones de configuración

5. Sistema de Análisis de Riesgo

5.1 Métodos de Adición de Contactos

5.1.1 Formulario de Diligencia Mínima

Proceso de diligencia mínima para evaluación básica de riesgo. Este formulario es ideal para contactos que requieren una evaluación rápida sin necesidad de documentación extensa. El sistema procesa automáticamente la información y genera un perfil de riesgo básico.
Formulario de diligencia mínima ubicado en Sitios → Formularios. Este formulario captura información esencial para realizar una evaluación de riesgo básica. Los campos están optimizados para capturar solo la información crítica necesaria para el análisis inicial.
  • Ubicación: Sitios → Formularios
  • Proceso: La completación del formulario activa webhook de procesamiento backend
  • Alcance: Solo evaluación de riesgo básica

5.1.2 Documentos de Diligencia Básica

Sección de documentos para diligencia básica. Esta área permite cargar y procesar documentos más detallados para una evaluación de riesgo completa. El sistema analiza automáticamente los documentos cargados y extrae la información relevante para el análisis de riesgo.
Tutorial detallado sobre cómo completar los documentos de diligencia básica. Este proceso permite una evaluación de riesgo exhaustiva en los 14 segmentos de negocio diferentes, proporcionando un perfil de riesgo completo y detallado.
  • Ubicación: Sección de Documentos
  • Proceso: La completación de documentos activa procesamiento backend
  • Alcance: Evaluación de riesgo integral en 14 segmentos de negocio

5.1.3 Carga Masiva de Contactos

Proceso de carga masiva de contactos para procesamiento en lote. Esta funcionalidad es esencial para empresas que necesitan procesar grandes volúmenes de contactos. El sistema valida automáticamente todos los campos requeridos antes de procesar la información.
  • Requisitos: Todos los campos requeridos deben estar mapeados correctamente
  • Procesamiento: Agregar etiqueta "enviar a procesar" para activar procesamiento backend
  • Validación: El sistema verifica campos requeridos faltantes antes de procesar

5.2 Algoritmo de Evaluación de Riesgo

5.2.1 Procesamiento Backend

  • Entrada de Datos: Información de contacto recibida de varias fuentes
  • Ejecución del Algoritmo: Backend procesa datos a través del algoritmo de evaluación de riesgo
  • Generación de Puntuación: El sistema calcula dos métricas clave:
    • Puntuación mínima
    • Riesgo inherente total
  • Asignación de Ubicación: Resultados devueltos a la ubicación correcta usando coincidencia de ID de ubicación

5.2.2 Determinación del Perfil de Riesgo

Demostración del proceso automatizado de determinación del perfil de riesgo. El sistema aplica lógica predefinida para categorizar los niveles de riesgo en Bajo, Medio o Alto. Esta categorización es fundamental para las decisiones de negocio y el cumplimiento regulatorio.

Lógica de Automatización: El sistema aplica lógica predefinida para categorizar niveles de riesgo:

  • Bajo
  • Medio
  • Alto

Alcance de la Evaluación:

  • Diligencia Básica: Riesgo determinado en 14 diferentes segmentos de negocio
  • Diligencia Mínima: Limitado a listas de verificación y evaluaciones de país

6. Repositorio y Gestión de Contactos

6.1 Repositorio de Cuenta Admin SL

  • Función: Repositorio central para todos los contactos procesados
  • Activador: Todos los contactos procesados se agregan automáticamente a la Cuenta Admin SL
  • Propósito: Mantiene base de datos integral de todos los contactos del sistema

6.2 Actualizaciones de Valores de Riesgo

  • Activador de Actualización: Cualquier modificación de valor de riesgo en cuentas de clientes
  • Ejecución de Webhook: Activa automatización en subcuenta
  • Sincronización Backend: Webhook enviado al backend para actualización de información de contacto
  • Actualización del Repositorio: Información de contacto actualizada en cuenta de repositorio
Flujo de actualización de valores de riesgo. Cuando se modifica cualquier valor de riesgo en las cuentas de clientes, el sistema automáticamente sincroniza estos cambios con el repositorio central, manteniendo toda la información actualizada en tiempo real.

6.3 Notificaciones de Contactos de Alto Riesgo

Sistema de notificaciones para contactos de riesgo medio y alto. Estas alertas automáticas aseguran que el equipo de gestión de riesgos sea notificado inmediatamente cuando se detectan contactos que requieren atención especial o seguimiento adicional.
  • Umbral de Riesgo: Contactos de riesgo medio o alto activan notificaciones
  • Proceso de Webhook: Envía webhook a cuenta Admin SL
  • Alertas por Email: Notificaciones automáticas por email enviadas a miembros del equipo
  • Contenido: El email incluye detalles de origen del contacto e información de riesgo

7. Directorio Público y Comunicación con Contactos

7.1 Integración del Directorio

URL: https://cdd.nilsdigital.com/
  • Fuente de Datos: La cuenta de repositorio alimenta el directorio público
  • Método de Búsqueda: El ID de contacto sirve como clave única de búsqueda
  • Funcionalidad: Recuperación en directorio público de información de contacto y perfiles de riesgo

El directorio público permite a los contactos y empresas verificar perfiles de riesgo de manera transparente. Esta funcionalidad es esencial para el cumplimiento regulatorio y la transparencia en las relaciones comerciales.

7.2 Comunicación Automatizada con Contactos

Sistema de comunicación automatizada que se activa al completar la evaluación de riesgo. Los contactos reciben automáticamente su información y un enlace directo a su perfil de riesgo, facilitando la transparencia y el acceso a la información.
  • Activador de Procesamiento: La completación de evaluación de riesgo activa flujo de comunicación
  • Contenido del Mensaje:
    • Resumen de información de contacto
    • Enlace directo al perfil de riesgo personal
  • Métodos de Entrega:
    • Mensaje de WhatsApp vía Savvy AI
    • Notificación por email
  • Cuenta Principal: Todas las comunicaciones procesadas a través de cuenta Admin SL

7.3 Sistema de ID de Contacto

  • Propósito: Identificador único para búsquedas en directorio
  • Uso: Los contactos pueden buscar sus perfiles usando su ID de contacto único
  • Beneficios: Acceso simplificado para individuos y empresas para verificar perfiles de riesgo

8.1 Chatbot Savvy AI

  • Función: Chatbot de soporte al cliente impulsado por IA
  • Entrenamiento: Equipado para manejar las preguntas y consultas más comunes
  • Escalamiento: Redirige consultas complejas al sistema de soporte humano

8.2 Pipeline de Soporte Humano

URL: https://savvysoporte.nilsdigital.com/inicia
Página de inicio del sistema de soporte humano. Los usuarios son redirigidos aquí cuando el chatbot Savvy AI no puede resolver sus consultas. La interfaz está diseñada para ser intuitiva y facilitar la comunicación rápida con el equipo de soporte.
Formulario de solicitud de soporte donde los usuarios detallan sus consultas o problemas. Este formulario captura toda la información necesaria para que el equipo de soporte pueda proporcionar asistencia efectiva y personalizada.
Pipeline de soporte donde se gestionan todas las solicitudes. Este sistema permite al equipo de soporte rastrear, priorizar y resolver eficientemente las consultas de los usuarios, asegurando que ninguna solicitud quede sin atender.
  • Activador de Escalamiento: Chatbot IA incapaz de resolver consulta

  • Adición al Pipeline: Contacto agregado automáticamente al pipeline de soporte para seguimiento
